1、数据库备份策略

1.1、数据库冷备份

概念：
       在固定的周期内，人为的将数据库中的数据进行备份，一般一式三份
缺点：

  1. 可能会造成数据丢失
  2. 如果数据量太多，则可能会导致备份时间很长。

ps: 虽然冷备份缺点很多，但是冷备份是恢复数据的最后手段，最好使用

1.2、数据库热备份


步骤：

  1. 当主库的数据发生变化时，会将更新的数据写入二进制日志文件中
  2. 从库中的IO线程会实时的监听主库的二进制日志文件，如果文件发生了变化，就会获取主库二进制文件中更新的记录，之后写入到从库的中继日志当中。
  3. 当中继日志中有数据时，从库会通过sql线程读取中继日志中的信息，之后会将数据进行同步
  4. 当上述操作完成后，主从同步完成，等待下一次同步

数据库进行主从备份时，一台主机可以配置n台从机，一般n为2—3.

1.3、数据库安装与配置

安装命令：rpm -ivh ..... .rpm
启动/停止/重启：service mysql start/stop/restart
设定用户名和密码：mysqladmin -u root password root

1.3.1 远程连接数据库

  1. (临时)关闭防火墙 service iptables stop
  2. 开启mysql对外访问权限 格式：GRANT ALL PRIVILEGES ON . TO ‘登录名‘@’登录ip,%为任意地址’ IDENTIFIED BY ‘登录密码’ grant all on *.* to 'root'@'%' identified by 'root';

ps: windows访问虚拟服务器 关闭Linux防火墙和开发MySQL对外访问权限，反之亦然。

       Ameba是一个以Mysql为底层数据存储，并对应用提供Mysql协议接口的proxy。它集中的响应应用的请求，依据用户事先设置的规则，将SQL请求发送到特定的数据库上执行。基于此可以实现负载均衡、读写分离、高可用性等需求。

分库分表

数据库垂直拆分

概念: 按照业务需求，将不同业务的表拆分到多个数据库中
原则: 如果表有关联关系.则不能拆分到不同的数据库中.

数据库水平拆分

概念: 将一张表中的数据拆分到多张表中

数据库优化策略

  1. “优化”sql语句.

    1. 尽量根据主键查询.
    2. 尽量使用单表查询.
    3. 如果进行关联操作时,尽量提早确定数据.减少笛卡尔积
    4. 表设计中如果业务复杂则可以进行反”三范式”

  2. 创建索引(原则:为频繁查询的字段创建索引.变动小的数据.)

  3. 使用缓存处理,减少数据执行次数. redis
  4. 定期进行数据转储.
  5. 分库分表 大数据量的表才是用分库分表.
